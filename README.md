# Vaya — Airport Mentions

A single-page React 19 app with a custom `@mention` autocomplete for airports. No mention libraries used — the mention logic is implemented from scratch, including pill-styled highlighting via a mirror overlay.

## Install & Run

```bash
npm install
npm run dev
```

Then open [http://localhost:5173](http://localhost:5173).

## Features

- **@mention autocomplete** — Type `@` followed by an airport name, city, country, IATA, or ICAO code to search ~5700 airports
- **Pill-styled mentions** — Completed mentions render as styled pills using a transparent mirror overlay
- **Airport preview panel** — A side panel displays airport details and a mini world map with a location marker alongside the suggestion dropdown
- **Recent airports** — Previously selected airports appear at the top of suggestions (persisted in `localStorage`)
- **Intro overlay** — Lottie-animated splash screen shown once per session
- **Undo support** — Uses `document.execCommand('insertText')` for native undo/redo compatibility
- **Keyboard navigation** — ArrowDown/ArrowUp to move through suggestions, Enter/Tab to select, Escape to close

## Mention Format

Selected airports are inserted as:

```
@✈ LAX — Los Angeles International Airport (Los Angeles, United States)
```

This format is matched by a Unicode regex pattern (`MENTION_PATTERN`) used for pill highlighting and mention detection.

## Mention Parsing Logic

1. On every input change, click, or caret movement, the app scans backwards from the caret to find a valid `@` trigger (must be at start of text or preceded by whitespace/punctuation).
2. The substring between `@` and the caret becomes the query. A newline cancels the mention.
3. The query filters airports via case-insensitive matching against name, city, country, IATA, and ICAO.
4. When a suggestion is selected, the `@query` text is replaced with the formatted mention and a trailing space.

## Dataset

Airport data sourced from [OpenFlights](https://github.com/jpatokal/openflights) (`airports.dat`). Two files are generated by `scripts/generate-airports.js`:

- `src/data/airports.json` — Full dataset (~6000 entries)
- `src/data/airports.curated.json` — Filtered to score >= 50 (~5700 entries, used by the app)

To regenerate:

```bash
npm run generate-airports
```

### Scoring & Ranking

Each airport has a pre-computed quality score for ranking suggestions:

| Factor                              | Points |
|-------------------------------------|--------|
| Has valid 3-letter IATA code        | +50    |
| Has valid 4-letter ICAO code        | +10    |
| Name includes "International"       | +20    |
| Major hub (top ~50 world airports)  | +40    |
| Major aviation country              | +5-15  |
| De-priority (Heliport, Air Base, Aerodrome, Seaplane, etc.) | -30 |

When the query is empty (`@`), top airports by score are shown. When typing, results sort by: exact IATA match > IATA starts-with > name/city starts-with > score descending > name alphabetically.
